# Criterios de aceptación

La entrega se considera lista únicamente cuando:

- comercio y administración crean pedidos completos;
- invitados no requieren cuenta;
- catálogo y tenant se validan server-side;
- totales y tarifa se recalculan;
- stock se actualiza de forma atómica;
- idempotencia evita duplicados;
- estados, asignación, auditoría, finanzas y notificaciones usan el flujo existente;
- snapshots preservan historial;
- móvil y escritorio son utilizables;
- pruebas, TypeScript, build, auditoría y migraciones pasan;
- preview y producción no registran errores;
- health check está correcto.
