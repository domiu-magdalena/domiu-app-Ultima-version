# Registro de riesgos

- Triggers financieros históricos pueden cambiar: cubrir con reconstrucción y prueba de pedido.
- Datos sin coordenadas usan tarifa base: señalizar al operador.
- Catálogo no tiene modificadores formales: conservar snapshot sin calcular valores no verificados.
- Reversión de esquema con datos sería destructiva: usar rollback de aplicación.
- Admin puede ejercer excepciones: auditar motivo y revisar periódicamente.
